# Lab 主页填坑指南

## 1. 页面改名

### 1.1 Research → Publication（原"研究"页改名为"发表"）
- 文件：`research/index.md`
- 改动：把 front matter 的 `title: Research` 改成 `title: Publication`，tooltip 也相应改

### 1.2 Projects → Research（原"项目"页改名为"研究"）
- 文件：`projects/index.md`
- 改动：把 front matter 的 `title: Projects` 改成 `title: Research`，tooltip 改

### 1.3 更新首页链接文字
- 文件：`index.md`
- 改动：
  - "See our publications" 按钮 → 指向 `publication`（但 link 仍是 `research`，因为文件夹没改名）
  - "Our Research" 特征块 → "Our Publications"
  - "Our Projects" → "Our Research"

> 注意：页面实际路径由文件夹名决定（`research/`、`projects/`），不改文件夹名的话链接不用动，只需改页面标题文字。

## 2. 基础信息配置（最先改）

**文件：`_config.yaml`**

```yaml
title: 你的实验室名称
subtitle: by 你的PI姓名
description: 实验室描述

links:
  email: 你的邮箱
  orcid: 你的ORCID编号
  google-scholar: 你的Google Scholar ID
  github: 你的GitHub用户名
  twitter: 你的Twitter账号
  youtube: 你的YouTube频道
```

---

## 3. 成员信息（PI/老师）

**文件夹：`_members/`**

每个成员一个 `.md` 文件，参考已有的 `jane-smith.md`。

主要字段：
- `name`：姓名
- `image`：照片路径（放 `images/` 下）
- `role`：`principal-investigator`（PI）、`postdoc`、`phd`、`master`、`undergrad`、`staff`
- `affiliation`：机构
- `links`：各种链接（个人主页、ORCID、GitHub、Twitter 等）
- 正文：个人简介（支持 Markdown）

### CV 处理
把 PDF 放到 `images/` 或 `assets/` 下，然后在 member 文件的 `links` 里加一行：
```yaml
links:
  cv: /assets/cv.pdf
```

---

## 4. 发表论文（Publication）

### 4.1 手动添加论文（最直接的方式）

**文件：`_data/citations.yaml`**

参考现有格式添加即可：

```yaml
- id: doi:10.xxxx/xxxxx
  title: 论文标题
  authors:
    - 作者1
    - 作者2
  publisher: 出版社
  date: '2024-01-15'
  link: https://doi.org/xxxxx
  image: https://...  # 论文配图（可选）
  type: paper
  description: 论文简短描述（支持 Markdown）
  tags:
    - tag1
    - tag2
```

其中 `id` 可以是 DOI、 PMID 或任意字符串。只要能唯一标识论文即可。

### 4.2 PDF 文件存放

把 PDF 放到 `images/` 或新建 `assets/` 目录，在 `citations.yaml` 里可以加 `pdf: /assets/xxx.pdf`（如果模板支持这个字段的话）。

## 5. 项目（Projects，现已改名为 Research）

**文件：`_data/projects.yaml`**

参考现有格式：

```yaml
- title: 项目名称
  subtitle: 副标题
  group: featured   # 有这个的是"精选"项目，会在页面顶部显示
  image: images/photo.jpg
  link: https://github.com/xxx
  description: 项目描述
  repo: github用户名/仓库名
  tags:
    - software
    - resource
```

## 6. 首页（index.md）

**文件：`index.md`**

需要改的内容：
- 标题 "Lab Website Template" → 实验室名称
- 整个"Highlights"区块的内容替换成你自己的

## 7. 团队页（Team）

## 10. 必做清单

| 优先级 | 事项 | 文件 |

| P0 | 添加 PI 成员信息 | `_members/xxx.md` |

## 文件夹/文件说明

| 路径 | 用途 |
|------|------|
| `_config.yaml` | 全站配置（标题、链接、导航顺序） |
| `_members/` | 成员信息，每个成员一个 .md |
| `_data/citations.yaml` | 论文列表 |
| `_data/projects.yaml` | 项目列表 |
| `research/index.md` | 发表论文页（Research → Publication） |
| `projects/index.md` | 项目页（Projects → Research） |
| `team/index.md` | 团队页 |
| `contact/index.md` | 联系方式页 |
| `index.md` | 首页 |
| `images/` | 所有图片放这里 |
| `blog/` | 博客页 |
| `_posts/` | 博客文章 |

---

## 注意事项

1. **不要改文件夹名**（`research/`、`projects/`），只改页面标题，这样所有内部链接都不用动。
2. 论文的 PDF 如果模板不支持直接引用，可以单独建一个 `assets/` 目录放 PDF，然后在论文描述里加下载链接。
3. 照片建议统一用 `.jpg` 或 `.png`，尺寸没有严格要求。
4. 每次改完本地跑 `bundle exec jekyll serve` 看效果。
